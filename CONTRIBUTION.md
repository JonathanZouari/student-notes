# Contributing to student-notes

This document introduces the people behind **student-notes** — a notes management app for students — and explains how you can join us as a contributor. Whether you want to fix a bug, suggest a feature, or improve the docs, this guide will get you started.

## How to Join and Contribute

We welcome contributions from fellow students and developers. To get started:

1. **Fork the repository** — click the *Fork* button at the top right of the [project page](https://github.com/RanHassid17/student-notes).
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/<your-username>/student-notes.git
   cd student-notes
   ```
3. **Create a feature branch** with a descriptive name:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** and commit them with a clear message:
   ```bash
   git commit -m "Add: short description of your change"
   ```
5. **Push the branch** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request** from your fork to the `main` branch of this repo. Please describe what you changed and why.
7. A maintainer will review your PR, suggest changes if needed, and merge it once it's ready.

If you're not sure where to start, check the [open issues](https://github.com/RanHassid17/student-notes/issues) or open a new one to discuss your idea before coding.
